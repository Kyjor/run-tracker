# HealthKit Integration Setup

This document explains how to complete the HealthKit integration for iOS.

## Overview

The HealthKit integration allows users to import running workouts from Apple Health into the app. The implementation consists of:

1. **Rust Commands** (`src-tauri/src/lib.rs`) - Tauri commands that can be called from the frontend
2. **Swift Bridge** (`src-tauri/Sources/HealthKitBridge.swift`) - Native iOS code to access HealthKit
3. **TypeScript Service** (`src/services/healthkitService.ts`) - Frontend service to handle imports
4. **UI** (`src/screens/SettingsScreen.tsx`) - Settings screen with import controls

## Required Steps

### 1. Add HealthKit Entitlements

Add HealthKit capability to your iOS app:

1. Open the Xcode project (generated by Tauri)
2. Select your app target
3. Go to "Signing & Capabilities"
4. Click "+ Capability"
5. Add "HealthKit"
6. Enable "HealthKit" in the capabilities list

### 2. Add Info.plist Permissions

Add the following to `Info.plist` (or via Xcode):

```xml
<key>NSHealthShareUsageDescription</key>
<string>We need access to your health data to import your running workouts.</string>
<key>NSHealthUpdateUsageDescription</key>
<string>We need access to update your health data with runs logged in the app.</string>
```

### 3. Link Swift Code to Rust

The Swift code needs to be compiled and linked with the Rust code. You'll need to:

1. Add the Swift file to your Xcode project
2. Create Rust FFI bindings to call the Swift functions
3. Update the build configuration

### 4. Alternative: Use Tauri Plugin

For a more maintainable solution, consider creating a Tauri plugin for HealthKit. This would involve:

1. Creating a plugin crate
2. Implementing the native iOS code in the plugin
3. Registering the plugin in your app

## Testing

Once integrated, test the following:

1. Request HealthKit permission
2. Import workouts from last 30/90 days
3. Verify duplicates are skipped
4. Check that imported runs have correct distance, duration, and metadata

## Notes

- The Swift code filters for running, walking, and hiking workouts only
- Duplicate detection uses distance and duration tolerance
- Heart rate data is included if available
- Workouts without distance are skipped
